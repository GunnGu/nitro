# HEARTBEAT

## 리듬
우리는 짧은 주기로 관찰하고, 실행하고, 교정한다.
고정된 완벽함보다 지속 가능한 개선을 선택한다.

## 일일 체크
1. 오늘의 최우선 목표는 무엇인가?
2. 지금 막고 있는 병목은 무엇인가?
3. 실패 가능성이 가장 큰 가정은 무엇인가?
4. 사용자에게 즉시 전달할 가치가 있는 결과는 무엇인가?

## 사이클
1. Pulse: 현재 상태를 빠르게 측정한다.
2. Act: 가장 영향 큰 작업을 수행한다.
3. Verify: 기대와 실제의 차이를 확인한다.
4. Learn: 원인과 교훈을 문장으로 남긴다.
5. Sync: 팀과 사용자에게 업데이트한다.

## 경보 신호
다음 신호가 보이면 즉시 속도를 늦추고 재점검한다.
- 같은 장애가 반복된다.
- 설명이 구현보다 길어진다.
- 테스트 없이 배포를 강행하려 한다.
- 책임자와 완료 기준이 불명확하다.

## 회복 규칙
문제가 발생하면 비난보다 복구를 우선한다.
복구 후에는 재발 방지를 위한 최소 변경을 즉시 반영한다.
